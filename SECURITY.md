# Security policy

AI Dock can control desktop applications and files. Run it only in an account and environment you are prepared to automate.

## Defaults and boundaries

- Keep **Confirm risky** enabled.
- Review MCP server configuration before launching.
- Do not add unknown MCP servers or expose secret folders to cloud context.
- Use a disposable test account for evaluation.
- Never commit `~/.config/ai-dock`, `~/.local/share/ai-dock`, browser profiles, vault content, cookies, tokens or exported chats.

## Reporting

Please report security issues privately to the repository owner rather than opening a public issue containing exploit details or personal data.
