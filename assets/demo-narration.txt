This is AI Dock, a natural-language desktop agent built for OpenAI Build Week. It turns informal requests into observable and recoverable computer workflows without treating model output as automatic authority.

The MCP page exposes one hundred eighty plus structured tools, adaptive cloud planning, confirmation for risky work, optional screen consent, task timelines, undo, persistent history, and immediate stopping.

The hamburger menu opens ChatGPT, Gemini, DeepSeek, HackerAI, local Qwen, Flow, MCP, and Control. Providers with restrictive embedded-browser policies, including Claude and Grok, connect through the separate browser popup while remaining available to workflows.

Flow can ask selected assistants in parallel or switch into Review Chain. The user chooses the exact order, moves providers up or down, saves reusable templates, resumes checkpoints, and synthesizes their perspectives.

The Control Center makes the system observable. Health, browser sessions, Connected Brain memory, power capabilities, durable agent tasks, recovery details, and regression tests stay visible in one place.

For private local chat, the Qwen page discovers installed Ollama models. Users can switch models and toggle thinking per question, while privileged MCP execution remains separately validated.

Cloud plans are untrusted advice, never direct shell code. Intent contracts, recursive schemas, authority checks, risk gates, crash-safe journals, action fingerprints, bounded recovery, and real world-state verification remain local.

Codex helped implement the runtime, inspect the real CachyOS and Hyprland environment, diagnose failures, and turn them into permanent tests. The human drove product direction, real-world testing, safety decisions, and acceptance criteria.

The sanitized public edition contains no credentials, browser profile, private chats, or vault data, and passes all fifty-four non-destructive checks. AI Dock shows that desktop agency can be intelligent, observable, resumable, and bounded.
